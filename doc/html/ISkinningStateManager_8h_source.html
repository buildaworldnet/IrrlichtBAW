14  LOCAL SPACE SKINNING:
15  Lets think about GPU Boning:
16  Need to calculate Hierarchy Positions Unless all Level 1+ bones use Global Skinning Space
17  Calculate by pingponging between buffer A and B, K times where K is the depth of the hierarchy of bones!
18  For XForm Feedback to work on that, it needs to output all of the joint hierarchy level data out at once for all bones.
19  Only Compute Shader/Pixel Shader with GL_LINES would be able to write to non-continous areas of memory.
20 
21  INTERMEDIATE REPRESENTATION:
22  [{Instance0_J0,Instance1_J0, ... , Instance1_J0},{Instance0_J1,Instance1_J1, ... , InstanceN_J1}, ... ,{Instance0_JM,Instance1_JM, ... , InstanceN_JM}]
23 
24  This would make it difficult/impossible to reference parent's data without queries if we were to start culling mid-bone update!!!!
25  Only Last Pass could possibly cull!
26 
27  +++LAST PASS NEEDS TO REORDER BONE MAJOR ORDER INTO INSTANCE MAJOR!!
28  Unless Skinning Shader can take that huge stride into account!!!! (bad for cache anyway)
29 
30 
31  1) GPU calculated Final Bone Positions, hence Final BBox for each bone => Final BBox for Node
32  2) So either readback or culling performed on GPU => Write to constrained location (compute/imgstore buffer or FBO) or xform feedback compaction
33  2a) Would need an additional compaction or index construction step to retrieve instance data (bones and stuff)
34  2b) XForm Feedback step compacts the buffer and disallows for using same buffer+pass for different sized aux data (different data per-bone)
35  This also requires the use of atomics to query the offsets and counts of instances for each instance-subtype (per meshes drawn with different materials)
36  ALSO THE INSTANCE-SUBTYPES NEED TO BE IN CONTINUOUS BUFFERS!!! - GRANULE ALLOCATION SCREWED!!!
37  Would effectively require a multi-granule buffer
38 
39  This makes the use of one ISkinningStateManager for all instances of crap using sourceHierarchy, very impractical although the GPU instructions remain the same!
40  FIRST SOLVE THE ISSUE OF MANY IMeshSceneNodeInstanced(s) USING THE SAME GPU CULLING PROGRAM AND XFMFEEDBACK LAYOUT TAKING MULTIPLE GPU CALLS TO CULL
41 
42  So the conclusion is that we need one ISkinningStateManager per IAnimatedSceneNode{Instanced}
43 
44 
45  3) To Obtain BBox for Instance
46  3a) Min/Max Blending on a 1D Texture Attached to FBO to which we write with GL_POINTS
47  3b) Min/Max Blending with atomics after "culling" don't write invisible bones
48  3c) Reverse Ping-pong to get child-inclusive bboxes
49  3d) During Instance Culling Pass read all N-bones per instance and get BBox that way (bad parallelism, more instances needed for GPU-saturation)
50 
51 
52  EVERY SINGLE BONE'S POSITION WOULD HAVE TO BE UPDATED if EJOUR_READ is ENABLED!!! AND READ BACK!!!
53 
54 
55  Given the picture we have here, GPU Boning is only a thing we want to consider if we're ever going to have 4096+ instances of the same LoD visible
56  (As of AMD Radeon R9 490X)
57 
58 
59  Hypothetical GPU-Boning Implementation:
60  1a) Fill Buffer LMB with Bone's Local TformMatrices if EJUOR_CONTROL
61  1b) Fill Buffer LMB by Compute/XFMFB/FBOPixelShader given a frame
62  IF HAS NON-GLOBAL SPACE BONES IN LEVELS DEEPER THAN 1:
63  2) Write to section of Buffer A reading from Buffer LMB for second bone hierarchy level with START shader
64  IF HAS NON-GLOBAL SPACE BONES IN LEVELS DEEPER THAN 2:
65  3) Write to section of Buffer B reading from Buffer A for third bone hierarchy level and so on pinponging between A and B with PASS shader
66  4) Write to Buffer A with UPDATE shader to calculate BBox and NormalMatrix , as well as join Buffer A and B together
67  (not needed if spec allows same buffer read via TBO range and XFMFB range)
68  5) Get Instance-wide BBox Perform Instance Culling!!!
69 
70  Since Local Skinning/Global Skinning switch is only available with Bones... GPU Boning always recomputes entire array!!!
71  GLOBAL SPACE SKINNING:
72  ONLY an OPTION with EBUM_READ or EBUM_CONTROL
73 
74  Only Reduces to updating IBoneSceneNode(s) and copying their absolute transformations over to the Output Array
75  GPU_BONING useless, calculating the inverses and bboxes less expensive than the GL call unless THOUSANDS of instances
